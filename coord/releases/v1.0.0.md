## What this is

xaxiu-harness is an internal-tool (Horizon C, NOT commercial product) multi-engine LLM dispatch + observability framework for solo-operator use. v1.0.0 is the first production tag — the install path is gated, the audit trail is forensic, the SDK has the surface we'll commit to as stable.

## Anti-patterns this release explicitly fights

1. **Don't trust untested install paths** → W13-INSTALL-VERIFY gates every PR.
2. **Don't build feature flags without an audit trail** → W13-AUDIT-JSONL is the foundation; every future auto-default lands a redacted row.
3. **Don't add new CLI verbs without explicit ROI** → `harness capabilities` introspection deferred 3 candidate verbs (`whoami`, `verbs`, `sdk-functions`).
4. **Don't merge `dispatch` + `review`** → kept separate per Round 2 dissent ruling.
5. **Don't build for hypothetical multi-user / plugin scenarios** → solo internal tool; multi-user / plugin-marketplace / VPS-hosted / best-of-N all explicitly dropped from scope.
6. **Don't ship features that hide what the harness is doing** → visible/overridable/auditable trio is the mandatory test for new behavior.

## Known issues at v1.0.0

- **Kimi engine HTTP 403** across all models (`kimi-for-coding`, `kimi-k2-0905-preway`, `kimi-latest`, `moonshot-v1-8k`). The release-gate panel ran without Kimi's voice because of this. Auth-side issue — `KIMI_API_KEY` is set + `harness capabilities` reports `keys_present.kimi=true`, but the upstream rejects every model. Not a v1.0.0 blocker (release surface is engine-agnostic; DeepSeek + MiMo voices were sufficient for the panel). Tracked as a fresh row to investigate post-release.
- **MiMo transient errors on parallel dispatch**: the panel hit `RemoteProtocolError` on MiMo in the parallel-3 ThreadPoolExecutor run; serial retry succeeded first attempt. Known pattern (W13-ENGINE-RETRY-RESILIENT identified this as a 12/12-retries-succeed transient). The retry is per-engine inside the streaming transport but doesn't cover the parallel-dispatch race. Out of scope for v1.0.0.

## How to bootstrap on a fresh clone

```
git clone <repo>
cd xaxiu-harness-standalone
pip install -e . --quiet && python -m harness today && python -m harness plan show
```

This one-liner was validated by sub-agent + real fresh session + CI test (`test_orientation_one_liner_works_post_install`) — all 3 independent signals.
