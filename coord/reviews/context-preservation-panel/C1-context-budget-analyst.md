# C1-context-budget-analyst

## 1. Headline recommendation

Default the agent’s dispatch result to a 200‑token summary (generated by the engine or a lightweight extractor) and store the full response in a retrievable cache — keeping 30‑dispatch sessions under 6K tokens of context growth from offloaded work, rather than 750K.

## 2. Concrete numbers / API signature / data shape

```python
@dataclass
class DispatchResult:
    success: bool
    summary: str                # NEW: default ~200 tokens, auto-generated
    text_ref: str | None = None # e.g. "cache://dispatches/abc-123" (populated always)
    dispatch_id: str
    engine_used: str
    tokens_used: int            # total engine tokens (in+out)
    tokens_in: int
    tokens_out: int
    cost_usd: float
    error: str | None = None

    @property
    def full_text(self) -> str:
        """Lazily fetch full response from cache (triggers I/O)."""
        return _cache_retrieve(self.text_ref)
```

New `dispatch_sync()` default parameters:

```python
def dispatch_sync(
    request: str | Dict[str, Any],
    response_format: str = "summary",   # "summary" | "full" | "both"
    summary_max_tokens: int = 250,       # cap for auto-summary
    cache_ttl: int = 3600,              # seconds to keep full text
    **kwargs,
) -> DispatchResult:
    ...
```

**Budget math:**

- 30 dispatches × 200 tokens (summary) = **6,000 tokens** → 3% of a 200K window.
- Same 30 dispatches at 25K full → 750,000 tokens → **375% of window**, impossible.
- Even 30 × 1K (very short responses) = 30K, still manageable but wasteful.
- Target for “30 dispatches under 100K growth” → average 3.3K/dispatch → default summary of 200–300 tokens is 15× smaller than that line.

## 3. The default that maximizes context preservation

`response_format="summary"` **must** be the default. The SDK’s `dispatch_sync()` silently returns a compact summary; the agent never sees the full text unless it explicitly opts in.

The library also auto-generates the summary using the engine itself (cheapest fast model) with a prompt like: *“Summarize the following response in at most N tokens. Preserve critical facts, actions, and numeric data.”* This adds < 100 extra tokens engine‑side and costs < $0.001 per dispatch.

For agents that just `from harness import dispatch_sync` and call it, they get:

```python
result = dispatch_sync("analyze the latest logs")
# result.summary -> "Found 3 anomalies in module X, severity high...\n"
# result.text_ref -> "cache://dispatches/d-456"
```

The agent can avoid calling `result.full_text` unless it truly needs the raw output.

## 4. The escape hatch

When the agent *does* need the full response (e.g., for detailed code output or diff), it has three mechanisms, all with explicit context cost:

| Method | Context cost | Notes |
|--------|-------------|-------|
| Set `response_format="full"` on the call | Full text returned directly in `DispatchResult.summary` (or `text`) – agent must decide to read it. | Simple, but agent pays the cost with awareness. |
| Call `result.full_text` lazily | Returns full text (no additional IO cost if cached). Agent can conditionally append `\n{result.full_text}` to prompt. | Clear code path; agent controls moment of inflation. |
| Set `response_format="both"` | Return both `summary` and `summary` field contains summary, and a new `raw_text` field (or `text`). | Doubles default cost: summary + full text. Explicit opt‑in required. |

Recommended safe pattern for an agent that *sometimes* needs full:
```python
result = dispatch_sync(prompt)
# ... use summary to decide ...
if need_more_detail(result.summary):
    chunk = result.full_text
    # inject only the necessary part (e.g., first 2000 tokens)
```

## 5. One risk / failure mode specific to the CONTEXT BUDGET ANALYST

**Summary truncation of critical information.**  
An auto‑generated summary may omit a single crucial number (e.g., “error rate 0.3%” vs “0.03%”), an exact file path, or a subtle instruction that the agent needs to act on. This could cause downstream reasoning failures that are hard to debug because the agent only sees the summary.

**Mitigation built into the design:**
- The summary prompt explicitly demands preservation of all numeric values, identifiers, and action items.
- The `text_ref` is always accessible; the agent can fall back to `full_text` at the cost of context.
- A future improvement: per‑field structured summaries (e.g., `summary = {"key_findings": str, "numbers": Dict, "actions": List[str]}`) to reduce information loss while keeping tokens low.
