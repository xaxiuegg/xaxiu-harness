## Closed schema (v1.2 HIGH-9)
The JSONL record MUST contain EXACTLY these keys, no more no less:
- `timestamp` (str, ISO 8601 UTC, generated by writer)
- `project` (str)
- `packet_path` (str)
- `backend` (str)
- `model` (str | None)
- `outcome` (str â€” one of the allowed values above)
- `latency_ms` (int)
- `fallback_to` (str | None)

`write_log_entry` MUST:
1. Validate `outcome` against the allowed set; raise `ValueError` if not.
2. Validate `backend` is in `SUPPORTED_BACKENDS` from `_constants`.
3. Validate `project` against `PROJECT_NAME_REGEX`.
4. Run `_redact()` on `packet_path` (it's a path, but could theoretically contain a key).
5. Build the dict with EXACTLY the 8 keys above (use generator expression or dict literal â€” do not loop and add keys).
6. Serialize via `json.dumps(record, ensure_ascii=True, sort_keys=True)`.
7. Append atomically: open file in `"ab"` mode + write bytes + fsync.

## `_redact(text: str) -> str` function

Replaces any of these patterns with `[REDACTED]`:
- `r"sk-[A-Za-z0-9]{20,}"` (Anthropic / OpenAI style keys)
- `r"Bearer\s+\S+"` (authorization headers)
- `r"(?i)api[_-]?key\s*[\"':=]+\s*[\"']?[A-Za-z0-9_-]{16,}"` (api_key= patterns)
- `r"ms-[A-Za-z0-9]{20,}"` (Kimi/Moonshot style keys)
- `r"deepseek-[A-Za-z0-9]{20,}"` (DeepSeek style keys)

Use `re.compile(...)` at module level for each pattern; iterate via single `for` loop. Return modified string. Never raises.
