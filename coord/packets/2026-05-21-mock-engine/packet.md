# Packet: MockEngine adapter for offline v2 end-to-end testing

## Mission

The v2 architecture has zero verified end-to-end paths because every test mocks `dispatch_packet`. We need a real engine adapter that returns deterministic stub responses so `harness coord run` can be exercised cold without burning Kimi credits, without API keys, and without network access.

New-file packet — disjoint from coord-wiring and engine-via-proxy.

## In-scope NEW files

- `src/harness/engines/mock.py` — `MockEngine` concrete impl + `MockResponseSet` for canned fixtures
- `tests/test_engines_mock.py`

## In-scope MODIFY files

- `src/harness/_constants.py` — add `"mock"` to SUPPORTED_BACKENDS
- `src/harness/engines/concrete.py::get_engine` — add `"mock"` branch
- `src/harness/engines/guards.py` — MockEngine output isn't subject to packet-trap guards (it's deterministic)

## MockEngine design

```python
# src/harness/engines/mock.py
import os
from typing import Any
from harness.engines.base import EngineResponse


class MockEngine:
    """Deterministic stub engine for offline testing.

    Reads canned responses from HARNESS_MOCK_FIXTURE_PATH (json file
    mapping prompt-substring -> response text) or falls back to a
    built-in echo response.
    """
    name = "mock"
    default_model = "mock-v1"

    def dispatch(
        self,
        packet_content: str,
        model: str,
        extra_args: dict,
    ) -> EngineResponse:
        # Look for fixture file
        fixture_path = os.environ.get("HARNESS_MOCK_FIXTURE_PATH")
        if fixture_path:
            try:
                import json
                fixtures = json.loads(Path(fixture_path).read_text(encoding="utf-8"))
                for key, response in fixtures.items():
                    if key in packet_content:
                        return EngineResponse(
                            success=True, text=response,
                            latency_ms=10, error=None,
                        )
            except Exception:
                pass
        # Built-in fallback: planner-style stub WavePlan with 2 trivial tasks
        if "WavePlan" in packet_content or "decompose" in packet_content.lower():
            return EngineResponse(
                success=True,
                text=_default_waveplan_stub(packet_content),
                latency_ms=10,
                error=None,
            )
        # Worker-style stub: FILE block
        return EngineResponse(
            success=True,
            text=_default_worker_stub(packet_content),
            latency_ms=10,
            error=None,
        )


def _default_waveplan_stub(packet: str) -> str:
    """Return a stub WavePlan JSON for the planner."""
    import json
    plan = {
        "schema_version": 1,
        "run_id": "MOCK-RUN-ID-PLACEHOLDER",  # planner overwrites
        "spec_path": "mock",
        "created_at": "2026-05-21T00:00:00+00:00",
        "planner_engine": "mock",
        "tasks": [
            {
                "worker_id": "worker-1",
                "title": "Mock task 1",
                "description": "Stub task for MockEngine smoke test",
                "read_set": ["README.md"],
                "write_set": ["mock-out-1.txt"],
                "test_set": [],
                "depends_on": [],
                "steps": [
                    {
                        "step_id": "s1",
                        "kind": "create",
                        "instruction": "Write hello to mock-out-1.txt",
                        "target_files": ["mock-out-1.txt"],
                        "expected_diff_lines": 1,
                        "required_tests": [],
                    }
                ],
                "estimated_kimi_minutes": 1,
                "max_context_tokens": 1000,
            }
        ],
        "integration_strategy": "squash",
        "notes": "Generated by MockEngine for offline test",
    }
    return f"```json\n{json.dumps(plan, indent=2)}\n```"


def _default_worker_stub(packet: str) -> str:
    """Return a stub FILE/REPLACE block for worker steps."""
    return """<FILE: mock-out-1.txt>
hello from MockEngine
</FILE>
"""
```

## Tests required

1. `MockEngine().dispatch("decompose this spec...", ...)` returns valid WavePlan JSON
2. `MockEngine().dispatch("worker step instruction", ...)` returns FILE block
3. With `HARNESS_MOCK_FIXTURE_PATH` set to a fixture file, prompt-substring matching works
4. `get_engine("mock")` returns MockEngine
5. `"mock" in SUPPORTED_BACKENDS`
6. `dispatch_packet(..., force_engine="mock")` end-to-end (mocked filesystem) returns successful DispatchResult

Target ≥5 new tests.

## Acceptance criteria

1. `harness coord plan --spec spec/x.md --engine mock` produces a valid (stub) plan.json
2. `pytest tests/ -q` green
3. Single commit: `feat(engines): MockEngine adapter for offline v2 end-to-end testing`
