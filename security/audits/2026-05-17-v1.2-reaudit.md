## Gaps (minor / non-blocking)

1. **MED-4 cosmetic issue**: The amendment text contains the author's internal monologue: *"CI guard: `! grep -rn 'os\.environ\[' src/` (we allow `os.environ.get()` but not bracket access? Actually we need to allow bracket for reading? Better: `! grep -rn 'subprocess.*os.environ' src/` plus a unit test."* This is meta-commentary that should not be in the final spec. The Section 3 CI guards block resolves the question by settling on `! grep -rn 'os\.environ\[' src/`, but the MED-4 amendment text itself should be cleaned up before merge. **Not blocking** — the final answer is unambiguous in Section 3.

2. **HIGH-2 cookie design**: The amendment uses `GET /api/login?token=...` to set the cookie. The audit recommended this same pattern, so this is faithful. However, putting the token in the URL means it appears in server access logs. The amendment's own HIGH-9 redaction does NOT specifically target query-string tokens. Not blocking (this matches the audit verbatim), but a v1.3 hardening opportunity.

3. **HIGH-5 scope creep**: The amendment introduces a `--purge-all` flag not requested by the audit. This is in line with LOW-8 backlog item ("Uninstall may miss DPAPI blobs outside `state/`"), so it's a coherent addition rather than scope creep. Acceptable — and the LOW-8 backlog entry correctly notes "already amended to cover `%APPDATA%\xaxiu-harness\` (HIGH-5)".

4. **MED-9 scope addition**: Amendment adds a new `/api/audit-log` endpoint listing routing changes. Audit recommended the SQLite table but didn't specify a query endpoint. The endpoint is a natural consequence of the table; acceptable.

5. **Implementation order — `_constants.py` ordering note**: Section 3 mentions `_constants.py` includes `SUPPORTED_BACKENDS` which is the set `{'csv', 'markdown', 'jira', 'linear'}` — this matches v1 §2 schema. However, LOW-6 (deferred) notes Jira/Linear credential handling isn't spec'd. If Jira/Linear backends are not implemented in Wave 2, `_constants.py` should perhaps gate them. Minor — not blocking since the constant just lists valid identifiers.
