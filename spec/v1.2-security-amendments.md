### Amendment for MED-2: `project_root` field has no path‑traversal validation
**Original spec location:** v1 §2 (line 47), v1.1 §1 (line 18), v1.1 §2.1 (line 32)  
**Original text:** `project_root | string | yes | Absolute path to project directory`  
**Replacement / insertion text:**  
Add pydantic validator to `AdapterConfig`:
```
@validator('project_root')
def validate_project_root(cls, v):
    p = Path(v).resolve(strict=True)
    # Must be a directory
    if not p.is_dir():
        raise ValueError("project_root must be an existing directory")
    # Must not be under Windows system directories
    forbidden_prefixes = [
        Path(os.environ.get('WINDIR', r'C:\Windows')),
        Path(os.environ.get('PROGRAMFILES', r'C:\Program Files')),
        Path(os.environ.get('PROGRAMDATA', r'C:\ProgramData')),
    ]
    for forbidden in forbidden_prefixes:
        if p.is_relative_to(forbidden):
            raise ValueError("project_root must not be under system directories")
    return p
```
**Verification check:**  
Unit test: `project_root="../../Windows/System32"` → resolve to `C:\Windows\System32` → `is_relative_to(forbidden)` true → ValidationError.

```yaml
- name: Security grep guards
  run: |
    echo "Checking forbidden patterns..."
    ! grep -rn 'yaml\.load(' src/ tests/
    ! grep -rn 'shell=True' src/
    ! grep -rn 'os\.environ\[' src/
    ! grep -rn 'host="0\.0\.0\.0"' src/
    ! grep -rnE 'execute\(["'"'"']\s*[A-Z]+.*\+' src/
    echo "All guards passed."
```
- The `os.environ` pattern catches both `os.environ['KEY']` and `os.environ.get` used inside string interpolation; but we allow `os.environ.get()` for reading env vars. Refine to `os\.environ\[` (only bracket access). Actually we want to ban `os.environ['KEY']` because it raises KeyError; `os.environ.get` is preferred.
